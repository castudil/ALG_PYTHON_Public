# ¿Qué es GitHub y por qué lo usamos?

Si es tu primera vez programando o participando en un curso de ciencias de la computación, es muy probable que términos como **Git** y **GitHub** te parezcan extraños. ¡No te preocupes! Esta breve guía te explicará qué son, para qué sirven y por qué son herramientas fundamentales en el mundo del desarrollo de software.

## 1. ¿Qué es Git?

**Git** es un **sistema de control de versiones**. Imagina que estás escribiendo un ensayo muy importante o un proyecto largo. Probablemente hayas guardado archivos con nombres como:
* `proyecto_final.docx`
* `proyecto_final_revisado.docx`
* `proyecto_final_AHORA_SI.docx`
* `proyecto_final_definitivo_v2.docx`

Git soluciona exactamente este problema. En lugar de crear múltiples archivos, Git guarda un "historial" cronológico de todos los cambios que haces en tu código a lo largo del tiempo. Si cometes un error o quieres volver a una versión anterior de tu trabajo, Git te permite "viajar en el tiempo" y recuperar esa versión de forma fácil y segura.

## 2. ¿Qué es GitHub?

Si Git es la herramienta instalada en tu computador para llevar el historial, **GitHub** es una plataforma de alojamiento en internet (la nube) que sirve para guardar y compartir esos historiales y proyectos.

* Piensa en **Git** como la aplicación de tu cámara de fotos.
* Piensa en **GitHub** como Google Photos o Instagram, donde subes las fotos para que tú y otras personas puedan verlas desde cualquier lugar.

En este curso, los profesores usarán GitHub para publicar el material de estudio, los laboratorios y los ejercicios (como si fuera un Google Drive o Dropbox, pero diseñado específicamente para los desarrolladores de software).

## 3. Conceptos Clave (El vocabulario del desarrollador)

A medida que uses GitHub, escucharás ciertas palabras con frecuencia:

* **Repositorio (o "Repo"):** Es simplemente la carpeta principal de tu proyecto. El curso que estás tomando utiliza un repositorio. Contiene todos los archivos, códigos, imágenes y documentos, además del historial completo de modificaciones.
* **Clonar (Clone):** Es el acto de descargar una copia exacta de un repositorio desde GitHub (en la nube) directamente a tu computador. Esto se hace solo la primera vez.
* **Commit:** Es el equivalente a "guardar los cambios" en Git. Un commit toma una "fotografía" de tu código en ese preciso instante y siempre va acompañado de un breve mensaje explicando qué cambiaste.
* **Push (Empujar):** Es el acto de "subir" o enviar tus nuevos *commits* (tus cambios guardados) desde tu computador hacia GitHub en la nube.
* **Pull (Tirar/Jalar):** Es el acto de descargar o "traer" los nuevos cambios desde GitHub hacia tu computador. Por ejemplo, cuando el profesor suba un nuevo laboratorio, ejecutarás un *Pull* para actualizar tu carpeta local inmediatamente.

## 4. ¿Por qué usamos GitHub en el curso?

1. **Es el estándar de la industria:** Prácticamente todas las empresas de tecnología en el mundo usan Git y GitHub (o plataformas similares como GitLab y Bitbucket). Aprender a usarlo desde tu primer año te dará una gran ventaja profesional invaluable.
2. **Facilita la distribución de material:** En lugar de enviarte archivos `.zip` cada semana por correo, descargar archivos sueltos del campus virtual, o tener problemas de compatibilidad, con un simple comando (`git pull`) puedes tener todo el material del curso sincronizado y actualizado al instante en tu propio computador.
3. **Colaboración moderna:** Permite que múltiples personas trabajen en el mismo código en equipo sin sobrescribir o eliminar por accidente el trabajo del otro.
